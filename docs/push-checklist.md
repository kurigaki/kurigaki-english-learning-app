# Push前チェックリスト

GitHubにpushする前に以下のチェックを実行してください。

---

## 必須チェック

すべてのチェックがパスしないとpushしないでください。

### 1. ビルドチェック

```bash
npm run build
```

**確認項目:**
- [ ] `Compiled successfully` が表示される
- [ ] ビルドエラーが発生しない

### 2. ESLintチェック

```bash
npm run lint
```

**確認項目:**
- [ ] `No ESLint warnings or errors` が表示される
- [ ] エラーや警告がない

### 3. TypeScript型チェック

```bash
npx tsc --noEmit
```

**確認項目:**
- [ ] 型エラーが発生しない（出力なし = 成功）

---

## 推奨チェック

品質向上のため、可能な限り実施してください。

### 4. 開発サーバーでの動作確認

```bash
npm run dev
```

**確認項目:**
- [ ] トップページ（`/`）が正常に表示される
- [ ] クイズ機能（`/quiz`）が動作する
- [ ] スピードチャレンジ（`/speed-challenge`）が動作する
- [ ] 単語詳細（`/word/[id]`）が表示される
- [ ] 学習履歴（`/history`）が表示される
- [ ] 実績一覧（`/achievements`）が表示される

### 5. レスポンシブ確認

- [ ] モバイル表示（375px）で崩れがない
- [ ] タブレット表示（768px）で崩れがない
- [ ] デスクトップ表示（1024px以上）で崩れがない

### 6. 音声機能確認

- [ ] 単語の発音が再生される
- [ ] 例文の読み上げが動作する

### 7. localStorage確認

- [ ] 学習記録が保存される
- [ ] ユーザーデータ（XP、レベル）が保存される
- [ ] 実績の解除が正常に動作する

---

## 一括チェックコマンド

以下のコマンドで必須チェックを一括実行できます：

```bash
npm run build && npm run lint && npx tsc --noEmit && echo "✅ All checks passed!"
```

## トラブルシューティング

### ビルドエラーが発生した場合

1. エラーメッセージを確認
2. 該当ファイルを修正
3. 再度 `npm run build` を実行

### ESLintエラーが発生した場合

```bash
# 自動修正可能なエラーを修正
npm run lint -- --fix
```

### 型エラーが発生した場合

1. エラーメッセージで示されたファイルと行番号を確認
2. 型定義（`src/types/index.ts`）を確認
3. 適切な型を付与して修正
