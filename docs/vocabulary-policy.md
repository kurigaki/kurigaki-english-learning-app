# Vocabulary Data Policy
> 最終改訂: 2026-03-31（品質原則に基づく方針追記）

## 1. ミッション

**「このコースを学習すれば、その試験に合格できる / 目標スコアに到達できる」**

- 英検○級コースを完了した学習者は英検○級に合格できる語彙力を身につける
- TOEIC○○○コースを完了した学習者はTOEIC○○○点を目指せる語彙力を身につける
- 中学○年コースを完了した学習者は定期テスト・高校受験に対応できる
- 高校○年コースを完了した学習者は定期テスト・大学受験に対応できる
- 会話コースを完了した学習者はそのレベルの英会話ができるようになる
- このアプリの存在価値は「英語学習の品質」にある。品質が低ければ普通のローグライクで遊べばいいだけになる

### 品質最優先の原則
- **meaningの正確さが最優先**。学習者が間違った意味を覚えることは絶対に避ける
- **語数の多さより試験頻出度**。語数が少なくても頻出語が揃っていれば合格できる
- **各コースは正しいソースから選定する**。英検データの安易な他コースへの転用は禁止
- **同じ語のmeaningはコース間で統一する**。英検の"go"と中学の"go"で意味が違ってはならない
- **meaning衝突はコード側で対応する**。meaningを歪めて衝突を解消してはならない
- 詳細は [quality-principles.md](quality-principles.md) を参照

### 絶対禁止事項
- 外部Webサービスからのスクレイピングデータの使用

- ライセンスが不明確なデータの使用

### 自作が必須のフィールド
- `meaning`（日本語の意味）: **全て自作**
- `example`（例文）: **全て自作**（AI生成 + 人間レビュー可）
- `exampleJa`（例文和訳）: **全て自作**
- カテゴリ分類: **独自基準**

### CC BY-SA 4.0 (ShareAlike) への対応
- 語彙リストデータ（単語リスト部分）は CC BY-SA 4.0 として切り離し管理する
- 意味・例文・カテゴリ等の独自作成データは自社著作物として管理する
- Credits画面およびデータファイル内にライセンス表記を含める

|--------|--------|

## 4. 単語データの必須フィールド

### 4.1 Word型（`words/*.js`）— 全フィールド必須

```typescript
{
  id: number,            // ユニークID（コース別ID範囲）
  word: string,          // 英単語（小文字統一。フレーズも可）
  meaning: string,       // 日本語の意味（自作。簡潔に10文字以内が理想）
  partOfSpeech: string,  // 品詞: "noun" | "verb" | "adjective" | "adverb" | "other"
  course: string,        // コース名
  stage: string,         // ステージ
  example: string,       // メイン例文（自作。クイズ出題時のデフォルト）
  exampleJa: string,     // メイン例文の和訳（自作。自然な日本語）
  examples: [            // 例文3件（必須。クイズ・ダンジョンでランダム出題に使用）
    { en: string, ja: string, context: string },  // 例文1（= example/exampleJaと同じ）
    { en: string, ja: string, context: string },  // 例文2（異なる場面）
    { en: string, ja: string, context: string },  // 例文3（異なる場面）
  ],
  difficulty: number,    // 難易度 1-7（course + stage から自動計算）
  category: string,      // 主カテゴリ（categories[0]と一致）
  categories: string[],  // 複数カテゴリ（必須。最低1つ）
  frequencyRank: number, // 頻度ランク（NGSL/CEFR-J等に基づく）
  source: string,        // データ出典（"CEFR-J" / "NGSL" / "TSL" / "original" 等）
}
```

### 4.2 例文3件のランダム出題仕様

各単語に**3件の例文**を用意し、クイズ・ダンジョンのクイズ戦闘で**ランダムに1件を選んで出題**する。

- `example` / `exampleJa`: メイン例文（1件目）。リスニング・ディクテーション等で使用
- `examples[]`: 3件の例文配列。`examples[0]` はメイン例文と同一
- クイズ出題時: `examples` からランダムに1件を選択して出題する
- 同じ単語でも出題ごとに異なる例文が表示されるため、学習の単調化を防ぐ
- 各例文は**異なる使用場面（context）**で作成し、単語の多面的な理解を促す
- `context` の値: "学校" / "家庭" / "会話" / "買い物" / "旅行" / "スポーツ" 等

### 4.2 ID範囲

| コース | ID範囲 |
|--------|--------|
| junior | 10000-19999 |
| senior | 20000-29999 |
| toeic | 30000-39999 |
| eiken | 40000-49999 |
| conversation | 70000-79999 |

### 4.4 充足率の目標

| フィールド | 目標 |
|-----------|------|
| example / exampleJa | **100%**（未設定は不許可） |
| examples[]（3件） | **100%**（全単語に3件必須） |
| categories[] | **100%**（最低1カテゴリ必須） |
| frequencyRank | **100%** |
| source | **100%**（データ出自の透明性担保） |

## 5. 例文品質基準

### 5.1 基本ルール

| 項目 | 基準 |
|------|------|
| 出自 | **全て自作**。外部辞書・教材からのコピー禁止 |
| 作成方法 | AI（Claude等）で候補生成 → 人間レビュー → 承認 |
| 長さ | 5〜12語。低ステージは短く、高ステージは長く |
| 文法レベル | 対象ステージのレベルに適合。上位ステージの文法は使わない |
| 語彙レベル | 例文中の他の語は対象ステージ以下のレベルのみ使用 |

### 5.2 自然さ・多様性

| 項目 | 基準 |
|------|------|
| 主語の多様性 | He / She / They / We / You / 人名を均等に使用。"I" は全体の15%以下 |
| パターン重複 | 同一構文テンプレート（I like X 等）は全体の10%以下 |
| 文脈の具体性 | 抽象的な文（"It is important."）より具体的な場面を描写 |
| ネイティブ自然さ | 文法的に正しくても不自然な表現は不可 |

### 5.3 和訳（exampleJa）ルール

| 項目 | 基準 |
|------|------|
| スタイル | です/ます調で統一 |
| 正確性 | 英文の意味と完全に一致すること |
| 自然さ | 直訳より意訳を優先。日本語として自然な文 |
| 英語混在 | 和訳文中に英語を混在させない |

### 7.2 frequencyRank

- **4**: 低頻度（TSL/BSL/NAWL / CEFR C1）
- **5**: 稀少（専門語彙 / CEFR C1以上）

## 8. 単語詳細画面（WordExtension）

### 8.1 表示セクション一覧

| # | セクション | 参照フィールド | 必須/任意 |
|---|-----------|---------------|----------|
| 1 | ヘッダー（単語・発音・品詞・意味） | word, meaning, pronunciation, partOfSpeech | 必須 |
| 2 | 記憶度 | （学習記録から計算） | 自動 |
| 3 | 例文（最大3件） | examples[] | 必須（最低1件） |
| 4 | コアイメージ | coreImage | 手書き優先→自動補完 |
| 5 | 関連語 | relatedWordEntries[] | 手書き優先→自動補完 |
| 6 | 使い方 | usage | 手書き優先→自動補完 |
| 7 | 類義語との違い | synonymDifferenceEntries[] | 手書き優先→自動補完 |
| 8 | 英英定義 | englishDefinition | 手書き優先→自動補完 |
| 9 | 語源 | etymology | 手書き優先→自動補完 |
| 10 | コラム | column | 手書き優先→自動補完 |

### 8.2 データ構成

```
src/data/
├── word-extensions/
│   ├── index.ts       ← 公開API（getWordExtension）
│   ├── manual.ts      ← 手書き拡張データ
│   └── generated.ts   ← 自動補完エンジン（手書きがない語のフォールバック）
```

### 8.3 手書き vs 自動補完

- **手書きデータは常に自動補完より優先される**
- 手書きがない語は `buildGeneratedExtension()` が最低限のセクションを提供
- 長期目標: 全語に手書きデータを提供する（品質最優先のため段階的に拡充）

## 9. 運用ルール

### 9.1 データ追加時のチェックリスト
- [ ] 同一コース内で `word` の重複がないか確認
- [ ] 継続コース（junior→senior, eiken各級）で前ステージとの重複がないか確認
- [ ] 全必須フィールドが設定されているか（充足率100%）
- [ ] 例文が品質基準（5.1〜5.3）を満たしているか
- [ ] カテゴリが語の実態に合っているか
- [ ] `source` フィールドにデータ出典が記録されているか
- [ ] 難易度がステージに対応しているか

### 9.2 禁止事項
- 外部サービスからのスクレイピング
- 他社教材の例文・定義のコピー
- ライセンスが不明なデータの使用
- 公序良俗に反する語彙の採用
- 同一コース内での単語重複（大小文字違いも同一語として扱う）

### 9.3 品質レビュー体制
- **ヘルパーT**: 出題品質の観点（クイズとして適切か）
- **マクロファージ**: カリキュラム設計の観点（レベル配置は適切か）
- **メモリーB**: 記憶定着の観点（覚えやすい例文か）
- **一般細胞**: 学習者視点（わかりやすいか）
