# Vocabulary Data Policy
> 最終改訂: 2026-04-08（frequencyTier実装に伴う語数方針改訂）

## 1. ミッション

**「このコースを学習すれば、その試験に合格できる / 目標スコアに到達できる」**

- 英検○級コースを完了した学習者は英検○級に合格できる語彙力を身につける
- TOEIC○○○コースを完了した学習者はTOEIC○○○点を目指せる語彙力を身につける
- 中学○年コースを完了した学習者は定期テスト・高校受験に対応できる
- 高校○年コースを完了した学習者は定期テスト・大学受験に対応できる
- 会話コースを完了した学習者はそのレベルの英会話ができるようになる
- このアプリの存在価値は「英語学習の品質」にある。品質が低ければ普通のローグライクで遊べばいいだけになる

### 品質最優先の原則
- **meaningの正確さが最優先**。学習者が間違った意味を覚えることは絶対に避ける
- **語数の多さより試験頻出度**。語数が少なくても頻出語が揃っていれば合格できる
- **各コースは正しいソースから選定する**。英検データの安易な他コースへの転用は禁止
- **同じ語のmeaningはコース間で統一する**。英検の"go"と中学の"go"で意味が違ってはならない
- **meaning衝突はコード側で対応する**。meaningを歪めて衝突を解消してはならない
- 詳細は [quality-principles.md](quality-principles.md) を参照

### 3.1.1 頻出度ティア（frequencyTier）

各単語に「頻出度ティア」を付与し、ユーザーのレベルに応じた出題を実現する。

| ティア | 名前 | 定義 | 対象ユーザー |
|--------|------|------|-------------|
| 1 | 頻出 | その試験/レベルで必ず出る語。合格/達成に必須 | 初学者（確実に合格したい人） |
| 2 | 標準 | よく出る語。高得点を目指すのに重要 | 一般学習者（おすすめ） |
| 3 | 発展 | 出ることがある語。満点/完璧を目指す人向け | 上級者 |

**ティア判定の参考基準（オープンソース優先）:**

| 判定項目 | 第1参考（データソース） | 第2参考（検証用） | 備考 |
|---------|----------------------|-----------------|------|
| レベル配置 | **CEFR-J Wordlist**（東京外大投野研究室、無料） | English Vocabulary Profile（Cambridge、無料検索） | CEFR-Jが日本の教育段階に直結 |
| 頻出度（一般） | **NGSL**（Browne et al.、CC BY-SA 4.0） | JACET 8000（大学英語教育学会） | NGSL上位語=ティア1 |
| 頻出度（TOEIC） | **TSL**（CC BY-SA 4.0） | 金のフレーズのスコア帯構造（検証用） | TSL上位語=TOEIC頻出 |
| 頻出度（学術） | **NAWL**（CC BY-SA 4.0） | — | 英検2級以上・高校上級 |
| 語数の妥当性 | **学習指導要領**（文科省、無料） | — | 中学1,600-1,800語、高校+1,800-2,500語 |

> **運用ルール**: 語の追加・削除・レベル移動・ティア変更は上記参考基準に照らして判断する。レビュアーの主観ではなく基準に基づく。商用書籍（金フレ、パス単、ターゲット1900等）は検証用として参照するが、データの直接転用はしない（著作権）。

### 3.3 コース間の単語共有（統合データモデル）

> **2026-04-08改訂**: データ統合（案A）により、1語1エントリで複数コースに所属するモデルに移行予定。

**統合後のルール:**
- **1語1エントリ**: 同じ word + partOfSpeech は1つのIDのみ存在する
- **courses配列**: 各エントリに所属コース・ステージの一覧を持つ
- **meaningは統一**: 同じ語は全コースで同じmeaningを使う
- **同一コース内のステージ重複は禁止**: eiken:5 と eiken:4 に同じ語は置かない
- **コース間での共有は推奨**: giveが中学・英検・会話に所属するのは正しい

## 4. 単語データの必須フィールド

### 4.1 MasterWord型（保存形式）— `master/level-{cefr}.json` に格納

> **2026-04-09改訂**: TSからJSONに移行。データとロジックを分離。

```json
{
  "id": 10735,
  "word": "interest",
  "meaning": "興味・関心・利子",
  "partOfSpeech": "noun",
  "examples": [
    { "en": "She has a deep interest in science.", "ja": "彼女は科学に深い興味を持っています。", "context": "学校" },
    { "en": "The bank offers a high interest rate.", "ja": "その銀行は高い利子率を提供しています。", "context": "金融" },
    { "en": "His talk sparked great interest.", "ja": "彼の話は大きな関心を呼びました。", "context": "社会" }
  ],
  "categories": ["daily", "finance"],
  "frequencyTier": 1,
  "courses": [
    { "course": "junior", "stage": "1", "meaning": "興味" },
    { "course": "eiken", "stage": "3" },
    { "course": "toeic", "stage": "500", "meaning": "利子" }
  ]
}
```

**meaning品質基準:**
- 文字数制限なし（辞書レベルの正確さを優先）
- 多義語は全主要語義を「・」区切りで記載
- コース別meaningはcourses配列の`meaning?`フィールドで設定
- master meaningにはcourse meaningの全語義が含まれること（整合性保証）
```

### 4.2 Word型（ランタイム型）— アプリが消費する型

```typescript
// MasterWord + コース固有フィールド
// getWordsForCourse() でコース別にフィルタ・展開した型
{
  // MasterWordの全フィールドを継承 +
  course: string,        // コースコンテキスト
  stage: string,         // ステージ
  meaning: string,       // コース別meaning（設定時）またはmaster meaning
  example: string,       // examples[0].en から導出
  exampleJa: string,     // examples[0].ja から導出
  difficulty: number,    // course:stage → DIFFICULTY_MAP（CEFR 6段階: 1-6）
  category: string,      // categories[0] から導出
  courses: [...],        // 全コース情報（word.coursesで他コース参照可能）
}
```

### 4.3 例文3件のランダム出題仕様

各単語に**3件の例文**を用意し、クイズ・ダンジョンのクイズ戦闘で**ランダムに1件を選んで出題**する。

- `example` / `exampleJa`: examples[0]から導出されるメイン例文
- `examples[]`: 3件の例文配列。`examples[0]` はメイン例文と同一
- クイズ出題時: `examples` からランダムに1件を選択して出題する
- 同じ単語でも出題ごとに異なる例文が表示されるため、学習の単調化を防ぐ
- 各例文は**異なる使用場面（context）**で作成し、単語の多面的な理解を促す
- `context` の値: "学校" / "家庭" / "会話" / "買い物" / "旅行" / "スポーツ" 等

### 4.4 ID採番ルール

**CEFRレベル × 10,000帯**で採番。IDからレベルを即座に特定可能。

| CEFR | ID範囲 | ファイル | 計算 |
|------|--------|---------|------|
| A1 | 1 - 10,000 | level-a1.ts | `Math.floor((ID-1)/10000) = 0` |
| A2 | 10,001 - 20,000 | level-a2.ts | `Math.floor((ID-1)/10000) = 1` |
| B1 | 20,001 - 30,000 | level-b1.ts | `Math.floor((ID-1)/10000) = 2` |
| B2 | 30,001 - 40,000 | level-b2.ts | `Math.floor((ID-1)/10000) = 3` |
| C1 | 40,001 - 50,000 | level-c1.ts | `Math.floor((ID-1)/10000) = 4` |
| C2 | 50,001 - 60,000 | level-c2.ts | `Math.floor((ID-1)/10000) = 5` |

- 各レベル内はアルファベット順でソート
- 品詞が違えば別ID（address:noun と address:verb は別）
- 語のCEFRレベルは courses 配列の最低難易度で決定

### 4.5 充足率の目標

| フィールド | 目標 |
|-----------|------|
| examples[]（3件） | **100%**（全単語に3件必須） |
| categories[] | **100%**（最低1カテゴリ必須） |
| frequencyTier | **100%** |
| courses[] | **100%**（最低1コース必須） |
| source | **100%**（データ出自の透明性担保） |

## 5. 例文品質基準

### 5.1 基本ルール

| 項目 | 基準 |
|------|------|
| 出自 | **全て自作**。外部辞書・教材からのコピー禁止 |
| 作成方法 | AI（Claude等）で候補生成 → 人間レビュー → 承認 |
| 長さ | 5〜12語。低ステージは短く、高ステージは長く |
| 文法レベル | 対象ステージのレベルに適合。上位ステージの文法は使わない |
| 語彙レベル | 例文中の他の語は対象ステージ以下のレベルのみ使用 |

### 5.2 自然さ・多様性

| 項目 | 基準 |
|------|------|
| 主語の多様性 | He / She / They / We / You / 人名を均等に使用。"I" は全体の15%以下 |
| パターン重複 | 同一構文テンプレート（I like X 等）は全体の10%以下 |
| 文脈の具体性 | 抽象的な文（"It is important."）より具体的な場面を描写 |
| ネイティブ自然さ | 文法的に正しくても不自然な表現は不可 |

### 5.3 和訳（exampleJa）ルール

| 項目 | 基準 |
|------|------|
| スタイル | です/ます調で統一 |
| 正確性 | 英文の意味と完全に一致すること |
| 自然さ | 直訳より意訳を優先。日本語として自然な文 |
| 英語混在 | 和訳文中に英語を混在させない |

| 5 | 関連語 | relatedWordEntries[] | 手書き優先→自動補完 |
| 6 | 使い方 | usage | 手書き優先→自動補完 |
| 7 | 類義語との違い | synonymDifferenceEntries[] | 手書き優先→自動補完 |
| 8 | 英英定義 | englishDefinition | 手書き優先→自動補完 |
| 9 | 語源 | etymology | 手書き優先→自動補完 |
| 10 | コラム | column | 手書き優先→自動補完 |

### 8.2 データ構成

```
src/data/
├── word-extensions/
│   ├── index.ts       ← 公開API（getWordExtension）
│   ├── manual.ts      ← 手書き拡張データ
│   └── generated.ts   ← 自動補完エンジン（手書きがない語のフォールバック）
```

### 8.3 手書き vs 自動補完

- **手書きデータは常に自動補完より優先される**
- 手書きがない語は `buildGeneratedExtension()` が最低限のセクションを提供
- 長期目標: 全語に手書きデータを提供する（品質最優先のため段階的に拡充）

## 9. 運用ルール

### 9.1 データ追加時のチェックリスト
- [ ] 同一コース内で `word` の重複がないか確認
- [ ] 継続コース（junior→senior, eiken各級）で前ステージとの重複がないか確認
- [ ] 全必須フィールドが設定されているか（充足率100%）
- [ ] 例文が品質基準（5.1〜5.3）を満たしているか
- [ ] カテゴリが語の実態に合っているか
- [ ] `source` フィールドにデータ出典が記録されているか
- [ ] 難易度がステージに対応しているか

### 9.2 禁止事項
- 外部サービスからのスクレイピング
- 他社教材の例文・定義のコピー
- ライセンスが不明なデータの使用
- 公序良俗に反する語彙の採用
- 同一コース内での単語重複（大小文字違いも同一語として扱う）

### 9.3 品質レビュー体制
- **ヘルパーT**: 出題品質の観点（クイズとして適切か）
- **マクロファージ**: カリキュラム設計の観点（レベル配置は適切か）
- **メモリーB**: 記憶定着の観点（覚えやすい例文か）
- **一般細胞**: 学習者視点（わかりやすいか）
