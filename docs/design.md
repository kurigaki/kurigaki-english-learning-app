### 2.2 音声読み上げ仕様

#### 自動再生

| 画面 | 問題タイプ | 自動再生 | 読み上げ内容 | 遅延 |
|------|----------|---------|-------------|------|
| クイズに挑戦 | 英→日 | ✅ | 英単語 | 300ms |
| クイズに挑戦 | 日→英 | ❌ | なし | - |
| クイズに挑戦 | 穴埋め | ✅ | 例文 | 300ms |

#### 手動再生（SpeakButtonコンポーネント）

- 全画面で音声ボタンによる手動再生を維持
- ブラウザ標準Web Speech APIを使用
- 再生中は重複再生を防止

### 2.3 イメージ画像表示仕様

#### 画像表示フォールバックチェーン

1. **単語固有URL** (`word.imageUrl`) - 単語専用の画像がある場合
2. **キーワードベースURL** (`word.imageKeyword`) - コンセプトマッピングから検索
3. **単語名検索** - 単語自体をコンセプトマッピングで検索
4. **カテゴリURL** - カテゴリに応じたUnsplash画像
5. **絵文字フォールバック** - 読み込みエラー時にカテゴリ絵文字＋グラデーション表示

## 2.6 画面遷移ルール（単語詳細への導線）

### 基本原則

**「単語が表示されている = その単語の詳細画面に遷移できる」**

アプリ内で単語が表示されるすべての画面において、その単語をタップすると単語詳細画面に遷移できるようにする。

### 遷移対応状況

| 画面 | 遷移箇所 | 状態 |
|------|----------|------|
| クイズリザルト | 回答した全単語リスト | ✅ |
| 学習履歴（概要タブ） | - | - |
| 学習履歴（苦手単語タブ） | 苦手単語リスト | ✅ |
| 学習履歴（履歴タブ） | 回答履歴リスト | ✅ |
| 苦手単語一覧 | 単語リスト | ✅ |
| スピードチャレンジリザルト | 正解/不正解単語リスト | 🔜 |
| 単語帳 | 単語一覧 | 🔜 |
| ホーム | 今日の単語など | 🔜 |

### 遷移パターン

```
/quiz（リザルト）      → /word/[id]?from=quiz     → /quiz（リザルトに復帰）
/history（苦手単語）   → /word/[id]?from=history  → /history
/history（履歴）       → /word/[id]?from=history  → /history
/weak-words           → /word/[id]?from=weak     → /weak-words
/speed-challenge      → /word/[id]?from=speed    → /speed-challenge
/word-list           → /word/[id]（予定）
```

#### `from`クエリパラメータの動作

| from値 | 戻り先 | ラベル |
|--------|--------|--------|
| quiz | /quiz（リザルト復元） | リザルトに戻る |
| speed | /speed-challenge | リザルトに戻る |
| history | /history | 学習履歴に戻る |
| weak | /weak-words | 苦手単語に戻る |
| (なし) | router.back() | 戻る |

### 3.2 記憶度の計算ロジック

```typescript
type MasteryLevel = "new" | "learning" | "familiar" | "mastered";

const getMasteryLevel = (accuracy: number | null, attempts: number): MasteryLevel => {
  if (attempts === 0 || accuracy === null) return "new";      // 未学習
  if (accuracy >= 80 && attempts >= 3) return "mastered";     // 習得済み
  if (accuracy >= 60) return "familiar";                       // 習得中
  return "learning";                                           // 学習中
};
```

## 6. 実績システム仕様

### 6.1 実績カテゴリ

| カテゴリ | 説明 | 例 |
|----------|------|-----|
| streak | 連続学習日数達成 | 3日連続、7日連続、30日連続 |
| score | 正解数達成 | 初めての正解、100問正解、1000問正解 |
| mastery | 単語習得数達成 | 10単語習得、50単語習得 |
| speed | スピードチャレンジ関連 | 初挑戦、10問正解、新記録達成 |
| special | 特別実績 | パーフェクト達成、コンボ達成 |

### 6.2 実績解除フロー

1. クイズ完了時に `storage.checkAndUnlockAchievements()` を呼び出し
2. 条件を満たす実績があれば自動解除
3. `AchievementUnlockPopup` でポップアップ表示
4. 実績一覧画面（/achievements）で確認可能

## 7. 一人プレイMVP時点での制約と前提

### 7.1 データ永続化

- **ストレージ**: localStorage（ブラウザ）
- **ユーザー識別**: 固定値`"default"`（認証なし）
- **データ移行**: 将来的にSupabase DBへ移行予定

### 7.2 認証

- **MVP**: 認証なし（一人プレイ前提）
- **将来**: Supabase Auth導入予定

### 7.3 画像・音声

- **画像**: コンセプトベース画像選択（単語名→キーワード→カテゴリの順でフォールバック）
- **音声**: Web Speech API（ブラウザ内蔵）
- **画像拡張**: 単語データに`imageUrl`/`imageKeyword`フィールドを追加可能

### 7.4 単語データ

- **現在**: `src/data/words.ts`に静的定義
- **将来**: DB管理、外部API連携予定

## 11. 将来拡張に向けた設計方針

### 11.1 データ拡張

- 単語詳細項目（コアイメージ、英英定義等）はプレースホルダーとして枠を用意
- 型定義に拡張フィールドを事前定義
- データ追加時にUIの変更不要

### 11.2 認証対応

- `userId`を全データに含めて設計
- localStorage → DB移行を想定した構造
- `storage.ts`の内部実装を変更するだけで移行可能

### 11.3 音声・画像

- 外部URL対応の設計（`imageUrl`, `audioUrl`フィールド）
- Web Speech APIをフォールバックとして維持

### 11.4 追加予定機能

| 機能 | 優先度 | 説明 |
|------|--------|------|
| 単語帳 | 高 | 全単語一覧表示・検索 |
| ブックマーク | 中 | お気に入り単語登録 |
| 復習リマインダー | 中 | 忘却曲線に基づく通知 |
| カスタムクイズ | 低 | カテゴリ・難易度選択 |
| ダークモード | 低 | テーマ切り替え |
